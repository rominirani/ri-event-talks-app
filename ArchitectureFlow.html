How it Works

  • Initial Load: The user visits the site, and server.js serves the static index.html along with its linked CSS and JS.
  • Data Fetching: Once main.js loads in the browser, it immediately makes an API request back to the server via fetch('/api/talks').
  • Server Response: The server reads the structured talk data from talks.json and sends it back to the client in JSON format.
  • Rendering & Interaction: The logic calculates the schedule timings, renders the talk cards, and handles the interactive modal whenever a user clicks on a talk. It also provides real-time filtering by Category and Speaker.
